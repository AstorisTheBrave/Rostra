## what this is

Every environment variable Rostra reads, and what each controls. Values live in a local `.env` (never
committed). Template is `.env.example`. All variables are defined/validated in `src/config.ts`.

## technical map

**Discord**
- `DISCORD_TOKEN` (required) — bot token
- `DISCORD_CLIENT_ID` (required) — application id (command registration)
- `OWNER_IDS` — comma-separated owner user ids
- `DEV_GUILD_ID` — guild for dev/owner-scoped command registration

**Runtime**
- `NODE_ENV` — development | production | test (default development)
- `LOG_LEVEL` — fatal | error | warn | info | debug | trace (default info)

**Database**
- `DATABASE_URL` (required) — Postgres URL; include `?connection_limit=5&pool_timeout=15`

**Cache / queues**
- `REDIS_URL` (optional) — enables shared Redis cache + BullMQ; in-memory fallback if unset

**Web**
- `PORT` (default 3000), `HOST` (default 0.0.0.0) — health endpoint + top.gg webhook

**Sharding**
- `SHARD_COUNT` (optional) — fixed shard count; otherwise auto
- `SHARDING_MODE` — native | hybrid (default native)

**AI (neutral names — provider is hidden)**
- `AI_API_KEY`, `AI_BASE_URL`, `AI_MODEL` (all optional) — used by the AI module only; never named after a vendor

**Music (Lavalink)**
- `LAVALINK_NODES` — JSON array of `{ id, host, port, password, secure }` (multi-node + failover)
- `LAVALINK_RECONNECT_TRIES` (5), `LAVALINK_RESUME` (true), `LAVALINK_REST_TIMEOUT` (10000)

**top.gg (optional)**
- `TOPGG_TOKEN` — stats autoposting; `TOPGG_WEBHOOK_AUTH` — vote webhook secret; `TOPGG_BOT_ID`

## gotchas

- Empty values are treated as unset (defaults apply). Required vars missing → boot fails with an aggregated error.
- Optional integrations no-op when their vars are absent.
- The local dev `.env` currently holds the 10 working Lavalink nodes (see `memory.md`, gitignored).
