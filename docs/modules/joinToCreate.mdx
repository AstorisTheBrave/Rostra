## what this is

"Join to create" voice channels: a member joins one designated hub voice channel and the bot instantly
makes them a personal temporary voice channel (which they control), then deletes it automatically when it
empties.

## technical map

**file paths**
- command: `src/modules/joinToCreate/index.ts`
- voice handler: `src/modules/joinToCreate/events.ts`
- config + temp-channel tracking: `src/modules/joinToCreate/service.ts` (+ `service.test.ts`)
- models: `prisma/schema.prisma` → `JoinToCreateConfig`, `J2CChannel`

**command** - `/voicehub` (Manage Server): `set <hub> [category]`, `nametemplate <text>`, `limit <n>`,
`disable`, `status`.

**event** - `voiceStateUpdate`: leaving an empty tracked temp channel deletes it; joining the hub creates
a `GuildVoice` channel (owner gets Manage/Move/Mute/Deafen over it), moves the member in, and records it.

**exports (service.ts)** - `getConfig/upsertConfig/invalidate`, `recordTemp/isTemp/removeTemp`,
`formatName(template, username)`.

## gotchas

- Requires the **Guild Voice States** intent (enabled).
- Temp channels are tracked in `J2CChannel` so cleanup survives restarts (any tracked channel that ends up
  empty is removed on the next relevant `voiceStateUpdate`).
- If moving the member into the new channel fails, the freshly created channel is deleted to avoid orphans.
- Requires `prisma migrate` for `JoinToCreateConfig`, `J2CChannel`.
