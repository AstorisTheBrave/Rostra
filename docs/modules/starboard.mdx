## what this is

Starboards: members react to a message with a star emoji and, once it passes a threshold, the bot reposts
it to a starboard channel. A guild can have **many** starboards, each with its own channel, star emojis, and
settings; a message can land on several at once. Also supports **auto-star channels** (the bot pre-reacts to
every new message) and a cross-board leaderboard. Built from the official Starboard bot docs (cloned into
`Archive/starboard-docs`).

## technical map

**file paths**
- command (board CRUD + edit + autostar + leaderboard): `src/modules/starboard/index.ts`
- reaction/autostar/delete engine + post builder: `src/modules/starboard/events.ts`
- board/autostar/entry data + pure helpers: `src/modules/starboard/service.ts`
  (+ `service.test.ts`, `decide.test.ts`)
- models: `Starboard`, `StarboardEntry` (one per board+message), `AutostarChannel`

**command** - `/starboard` (Manage Server): `create <name> <channel> [emojis]`, `delete <name>`, `list`,
`edit <name> [required_stars] [remove_stars] [emojis] [self_star] [filter_bots] [sync_deletes]
[reward_role] [reward_stars] [author_role] [enabled]`, `ignore <name> <channel>`, `leaderboard`, and the
`autostar add|remove|list` group. `create` flips the tenant `starboard` flag.

**routing (events.ts)** - on a star reaction, `boardsForEmoji` finds every enabled board whose `emojis`
include the reacted emoji; for each, the message is filtered (skip if in the board's `ignoredChannels`, if
its author lacks `authorRoleId` for a role-based board, or if it lives in any board's channel), the valid
star count is computed per board (`effectiveStarCount` honouring `selfStar`/`filterBots`), and
`decideStarboard(stars, requiredStars, removeStars, hasPost)` returns `post|keep|remove|none`. `post` grants
the reward role (`earnsReward`). Auto-star channels are handled on `messageCreate` (the bot reacts with the
channel's emojis; it does **not** auto-post - the message must still earn its stars). `messageDelete` removes
each board's post when that board has `syncDeletes`.

**exports (service.ts)** - board CRUD (`listBoards`/`getBoard`/`createBoard`/`updateBoard`/`deleteBoard`,
cached per guild) + `boardsForEmoji`; autostar CRUD (`listAutostar`/`getAutostar`/`addAutostar`/
`removeAutostar`); entry ops (`getEntry`/`upsertEntry`/`entriesForMessage`/`deleteEntriesForMessage`);
`topStarred` (groupBy across all boards); and pure, tested helpers `parseStarEmoji`, `parseEmojiList`,
`reactionKey`, `emojiDisplay`, `effectiveStarCount`, `decideStarboard`, `earnsReward`.

## gotchas

- Multi-board: emoji-based routing is inherent (different emoji -> different board); role-based and
  channel-based routing are the `authorRoleId` and `ignoredChannels` filters. A message can appear on
  multiple boards simultaneously (one `StarboardEntry` per board+message).
- `removeStars` is the hysteresis floor (null => `requiredStars`, i.e. no hysteresis). `filterBots` ignores
  bot authors/reactions; `selfStar` lets authors star their own messages.
- Valid star counts come from fetching the reaction's users, so very large counts cap at Discord's 100-user
  fetch.
- Needs `GuildMessageReactions` intent + `Partials.Reaction`/`Partials.Message` (set on `BotClient`).
  Gated on the tenant `starboard` flag.
- **Not yet built** (the deepest Starboard-bot layer): the override-inheritance engine (per-context setting
  overrides that inherit from a base), blacklists/whitelists, message-filter groups with display tiers, and
  downvote emojis. The multi-board + per-board-settings + autostar + reward + leaderboard core is complete.
