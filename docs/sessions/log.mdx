### 2026-06-05 (later - safeAck core utility)

**changed**: Added `src/utils/safeAck.ts` (`withSafeAck`) - auto-defers any interaction whose handler
hasn't replied within ~2.3s (margin under Discord's 3000ms deadline), showing the native "thinking…" state

`SlashCommand`/`ComponentHandler`. 70/70 tests (3 new); typecheck + lint + undercover clean; 19 commands.

**files touched**: `src/utils/safeAck.ts` (+ test), `src/pipeline/runCommand.ts`,

### 2026-06-05 (later - safeAck heartbeat + V2 follow-up fallback)

**changed**: Added an opt-in `heartbeat` to `withSafeAck` - for long-running handlers it edits the
deferred reply every ~8s ("⏳ Rostra is still working on it… (Ns)"), and a graceful note as the ~15-min
interaction token nears expiry; stops the instant the handler finishes. Heartbeat uses Components V2 to
stay flag-consistent with the final reply. Added `heartbeat?` to `SlashCommand`/`ComponentHandler`
(passed through pipeline + router); enabled on `/ask`. **De-risked the deferred→V2 path**: the components
kit's `send()` now falls back to a fresh `followUp` if an `editReply` with the IsComponentsV2 flag is ever
rejected (Discord can refuse toggling that flag on edit), so auto-deferred commands never fail to render.
72/72 tests (2 new heartbeat tests); typecheck + lint + embed + undercover clean; 19 commands.

**files touched**: `src/utils/safeAck.ts` (+ test), `src/utils/components.ts`, `src/types/module.ts`,
`src/pipeline/runCommand.ts`, `src/interactions/router.ts`, `src/modules/ai/index.ts`.

**verify-on-live note**: confirm `deferReply()` → `editReply({ flags: IsComponentsV2, components })` renders
